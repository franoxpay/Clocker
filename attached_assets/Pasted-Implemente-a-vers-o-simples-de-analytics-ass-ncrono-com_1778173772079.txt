Implemente a versão simples de analytics assíncrono com fire-and-forget, sem BullMQ.

Faça exatamente isto:

1. Procure todos os redirects onde existe await antes do res.redirect relacionado a:
- createClickLog
- incrementOfferClicks
- daily metrics
- analytics
- logs não essenciais

2. Não altere a lógica de decisão black/white.

await storage.createClickLog(...)
await storage.incrementOfferClicks(...)

Em algo assim:

storage.createClickLog(...).catch(err => {
  console.error('[Analytics] createClickLog failed:', err.message)
})

storage.incrementOfferClicks(...).catch(err => {
  console.error('[Analytics] incrementOfferClicks failed:', err.message)
})

return res.redirect(302, targetUrl)

7. Garanta que nenhuma promise fique sem catch.
