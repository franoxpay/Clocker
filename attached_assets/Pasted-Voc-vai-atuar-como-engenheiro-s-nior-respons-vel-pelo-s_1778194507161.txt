- página de conta suspensa.

REGRAS FINAIS DE NEGÓCIO

1. Cada plano possui limite mensal de clicks.

2. O usuário pode ultrapassar até 20% do limite sem upgrade imediato.

Exemplo:
Plano Básico:
30.000 clicks

Tolerância:
36.000 clicks

Até 36k:
- continua funcionando normalmente;
- apenas registrar aviso interno se necessário.

3. Quando ultrapassar os 20%:
- registrar excedente;
- iniciar cobrança de clicks extras;
- iniciar tentativa de auto-upgrade para o próximo plano compatível.

4. O auto-upgrade deve:
- acontecer apenas para o próximo plano acima;
- nunca pular múltiplos planos;
- nunca fazer downgrade automático.

5. Ordem oficial dos planos:
- Plano Básico
- Plano Avançado
- Plano Pré-Escala
- Plano Escala
- Plano Ilimitado

6. Plano Free:
- não participa de auto-upgrade automático.

7. Plano Ilimitado:
- nunca cobra excedente;
- nunca faz upgrade.

8. Se o pagamento falhar:
- iniciar grace period de 3 dias;
- manter redirects funcionando normalmente durante esses 3 dias;
- registrar status grace_period;
- enviar notificações/email;
- mostrar aviso no painel.

9. Após 3 dias sem regularização:
- marcar usuário como suspended;
- bloquear funcionalidades;
- impedir uso do sistema;
- redirects NÃO devem retornar 404;
- redirects devem enviar para página de conta suspensa.

10. Criar rota simples:
GET /suspended

Essa página deve:
- ser extremamente leve;
- HTML simples;
- sem React;
- sem auth;
- sem dependência pesada;
- sem stack trace;
- sem dados técnicos.

Texto simples:
"Conta temporariamente suspensa. Regularize sua assinatura para reativar o acesso."

11. Quando usuário suspended acessar redirect:
- redirecionar para /suspended.

12. Quando pagamento for regularizado:
- remover grace period;
- remover suspensão;
- restaurar redirects automaticamente;
- restaurar acesso ao painel.

13. NÃO deletar:
