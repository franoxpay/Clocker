2. Backend de e-mails

- Resend
- sendEmail
- email templates
- email_logs
- transactional emails
- subscription reminders
- password reset
- billing emails
- suspension emails

3. Templates
Verificar:

- se há teste de envio;

4. Logs de e-mails
Verificar:

- subject;
- template;
- status;
- provider message id;
- erro;
- createdAt;
- opened/clicked, se existir;
- se existe retry;
- se existe paginação/filtros.

5. Envio e confiabilidade
Verificar:
- o que acontece se Resend falhar;
- se erro é logado;
- se existe retry;
- se existe fila;
- se email crítico pode ser perdido;
- se envio é síncrono ou assíncrono;
- se email pode atrasar resposta da API;
- se existe rate limit.

6. Segurança
Verificar:
- se dados sensíveis aparecem nos logs;
- se HTML pode ser injetado por template;
- se admin pode enviar email para qualquer endereço;
- se existe risco de spam;
- se existe validação de destinatário;
- se existe abuso de envio manual;
- se tokens aparecem em logs.

- pagamento falhou;
- assinatura cancelada;
